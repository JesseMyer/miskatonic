# Miskatonic

A governed AI platform for enterprise architecture practice.

Named for the institution that governs access to forbidden knowledge:
it maintains the collection, produces the practitioners, and ensures
what is retrieved can be used responsibly.

**github.com/JesseMyer/miskatonic**

---

## What This Is

Miskatonic is not a template library. It is not a chatbot wrapper.
It is a structured knowledge platform for enterprise architecture
practice: altitude-aware, domain-calibrated, TOGAF-metamodel-conformant,
and governed at every layer.

Every artifact declares its altitude, domain, state, metamodel constructs,
parent pattern, and qualification holds. Nothing floats free. Every risk
maps to a broken metamodel construct relationship. Every finding triggers
a Bureau suggestion. Every qualification hold gates descent to the next
altitude.

Zero human-written code. The platform is governed by Jesse Myer and
generated by Claude. This is the proof of concept.

---

## What It Does

When two companies merge, they suddenly have two of everything: two
customer systems, two finance systems, two IT departments, two sets of
rules. Nobody agrees on which ones to keep. Billions get spent. Years go
by. Half of mergers still fail.

Not because the technology is hard. Because nobody drew the map before
they started digging.

Miskatonic draws the map. It produces the same governed architecture
artifacts a senior enterprise architect would produce after decades of
M&A engagements: altitude-calibrated patterns, structured prompts,
scored findings, roadmaps, and a governed record of every decision made
along the way.

It does not replace the architect. It arms one.

Mergers are the opening domain. The same platform applies to any hard
technology decision an organization faces: surviving a data center
outage, picking the right cloud, evaluating vendors, proving to a
regulator that your technology is under control.

## Architecture Principles

**Altitude discipline.** Every artifact declares its altitude. Five
levels: System, Subsystem, Component, Sequence, Schema. Cross-altitude
coherence is verified. Altitude confusion is a Bureau finding.

**TOGAF full metamodel, strict adherence.** Every artifact declares its
governing metamodel constructs. Every risk maps to a broken construct
relationship. ADM is a light optional overlay, not a governing framework.

**Domain calibration.** Every artifact is calibrated to a specific EA
engagement domain. Generic EA is decoration.

**Dual-track motivational layer.** Elicited aspirational intent and
consumed curated documentation are separate records. The delta between
them is a Bureau finding, not a reconciliation target. Both tracks are
preserved.

**Bureau suggests, not just scores.** Every finding carries a
best-practice remediation suggestion, scoped to the declared region and
Ossuary reference tier. The architect disposes. Grimoire records.

**Region-aware.** Every engagement declares a region. The Ossuary
reference tier routes to the appropriate framework, analyst, and market
signal sources for that region. Gartner is not authoritative in APAC.
The platform knows this.

**Conditional construct typing.** Construct declarations are gated by
Vril qualification prompts. No construct is typed without qualification
where qualification is required. A qualification hold is not a finding
until its resolution deadline is exceeded.

**Zero human-written code.** Jesse Myer governs and calibrates. Claude
generates and assembles.

## The Vril Prompt Library

Vril prompts invoke Ossuary. They are the instruments through which a
practitioner engages the platform. Every prompt declares its altitude,
output type, and Ossuary dependency.

Six output types per altitude:

| Output Type | Purpose |
|---|---|
| Narrative | Governing prose at altitude: the 30,000-foot view, the domain decomposition, the structural description |
| Diagram | ArchiMate or UML artifact at altitude: capability map, application layer, process flow, data model |
| Scoring | Bureau assessment against target state benchmark: 0-4 dimension scoring, findings, suggestions |
| Roadmap | Pelagic wave structure: entry conditions, exit criteria, Bureau checkpoints, dependency matrix |
| Finding | Formal Bureau finding record: severity, broken construct, remediation, escalation, Grimoire entry |
| Qualification | Vril qualification prompt: gates construct typing before pattern closure at any altitude |

## The Ossuary Reference Tier

Three sub-tiers routing by declared region:

**Tier 1: Framework.** TOGAF, SABSA, COBIT, ISO 22301, NIST, 23 NYCRR 500,
DORA, FedRAMP. Governed, stable, highest citation authority.

**Tier 2: Analyst.** Gartner, Forrester, IDC, regional equivalents.
Date-stamped, region-weighted, 24-month shelf life.

**Tier 3: Market Signal.** Crunchbase, SEC filings, regulatory filings.
Always paired with Tier 1 or Tier 2. Never standalone.

No Bureau suggestion cites an undeclared reference.

## Governance Decisions on Record

Seven platform-level governance decisions locked at Phase 1 close:

1. **Dual-track motivational layer.** Elicited and consumed records are
   separate artifacts. Delta is a Bureau finding, not a merged record.

2. **Org structure as-reported at current state.** Contradictions surface
   at Subsystem altitude as findings. As-reported is a metamodel signal,
   not a correction target.

3. **Conditional construct typing.** Vril qualification prompts gate
   construct declaration before pattern closure. A hold is not a finding
   until its deadline is exceeded.

4. **Bureau suggests, not just scores.** Every finding carries a
   best-practice proposal. The architect disposes. Grimoire records.

5. **Strict TOGAF full metamodel adherence.** ADM is a light optional
   overlay only. The full metamodel governs every artifact at every
   altitude.

6. **Ossuary reference tier with region as routing key.** Three sub-tiers.
   Region declared per engagement. API integration layer deferred to
   Phase 4.

7. **Compliance mapping deferred to Phase 4.** Not in scope for M&A
   Pattern corpus. Phase 4 domain expansion introduces it.

## Platform Standards on Record

Standards that emerged from the M&A corpus and are now platform-enforced,
not engagement-preference:

- **Payroll parallel run minimum.** No payroll platform decommissioned
  without minimum two validated parallel pay cycles.
- **Taxonomy mapping threshold.** 80% canonical taxonomy mapping required
  at Wave 3 entry for Product Domain. Below threshold blocks Wave 3.
- **CMDB completeness thresholds.** 85% at Wave 2, 95% at Wave 3.
  Below threshold, change impact analysis is not reliable.
- **Parallel close validation.** Minimum two periods automated
  consolidation validated against manual before manual is retired.
- **Finance and HCM SME sign-off.** Account code mapping and
  compensation element mapping require domain SME review. Not an
  ARB function alone.
